## 构建镜像本地调试

修改镜像构建文件，先拉取go语言依赖，利用docker缓存机制，设置加速，设置启动命令。

拉取的镜像从本地开发镜像仓库拉取，加快镜像构建速度。

### 服务端调试

#### 构建服务端镜像

```bash
docker buildx build --file Dockerfile-frps \
--platform linux/amd64 --tag firfe/frps:0.69.1 \
--output type=docker .
```

#### 创建必要文件

```bash
mkdir -p ts-调试/frps/配置 && touch ts-调试/frps/配置/frps.toml && touch ts-调试/frps/compose.yaml
```

##### 配置文件 frps.toml

```bash
cat <<EOF > ts-调试/frps/配置/frps.toml
bindPort = 7000

[auth]
method = "token"
token = "frps"

[webServer]
addr = "0.0.0.0"
port = 7500
#assetsDir = "/static"

EOF
```

##### 部署文件 compose.yaml

```bash
cat <<EOF > ts-调试/frps/compose.yaml
name: frps
services:
  frps:
    container_name: frps
    image: firfe/frps:0.69.1
    network_mode: host
    tty: true
    stdin_open: true
    volumes:
      - ./配置:/frp
      - ?/代码/web/frps/dist:/static

EOF
```

#### 运行命令

- 启动
  ```bash
  docker compose -f ts-调试/frps/compose.yaml up -d
  ```
- 启动新构建镜像
  ```bash
  docker compose -f ts-调试/frps/compose.yaml up -d --force-recreate
  ```
- 重启
  ```bash
  docker compose -f ts-调试/frps/compose.yaml restart
  ```
- 删除
  ```bash
  docker compose -f ts-调试/frps/compose.yaml down
  ```

### 客户端调试

#### 构建服务端镜像

```bash
docker buildx build --file Dockerfile-frpc \
--platform linux/amd64 --tag firfe/frpc:0.69.1 \
--output type=docker .
```

#### 创建必要文件

```bash
mkdir -p ts-调试/frpc/配置 && echo -e "{}" > ts-调试/frpc/配置/frpc_store.json && touch ts-调试/frpc/配置/frpc.toml && touch ts-调试/frpc/compose.yaml
```

##### 配置文件 frpc.toml

```bash
cat <<EOF > ts-调试/frpc/配置/frpc.toml
serverAddr = "192.168.13.7"
serverPort = 7000

user = "调试"
clientID = "4"

[transport]
wireProtocol = "v2"

[auth]
method = "token"
token = "frps"

[webServer]
addr = "0.0.0.0"
port = 7400
#assetsDir = "/static"

[store]
path = "/frp/frpc_store.json"

EOF
```

```bash
cat <<EOF > ts-调试/frpc/compose.yaml

```

#### 运行命令

## 构建 Win 版本

- 创建容器
  ```bash
  docker run --rm -it --name KaiFa_exe -h KaiFa_exe --restart=no \
  --cpus=1 --memory=2G --network host \
  -e VERSION=0.69.1 -e CGO_ENABLED=0 -e GOOS=windows -e GOARCH=amd64 \
  -v ./ts-调试/go-mod:/go/pkg/mod -v ./ts-调试/go-cache:/root/.cache \
  docker.code.firfe.work/golang:1.26 bash
  ```
- 复制文件 容器外执行
  ```bash
  docker cp ./代码 KaiFa_exe:/app
  ```
- 设置环境变量
  ```bash
  go env -w GOPROXY=https://mirrors.aliyun.com/goproxy/,direct && \
  LDFLAGS="-s -w -X github.com/fatedier/frp/pkg/util/version.Version=${VERSION}" && \
  cd /app
  ```
- 编译服务端
  ```bash
  go build -v \
  -trimpath \
  -ldflags "${LDFLAGS}" \
  -tags frps \
  -o ./release/frps_windows_amd64_${VERSION}.exe \
  ./cmd/frps
  ```
  - 从容器中复制编译后的软件到本地 容器外执行
    ```bash
    docker cp KaiFa_exe:/app/release/frps_windows_amd64_0.69.1.exe ./ts-调试/
    ```
- 编译客户端
  ```bash
  go build -v \
  -trimpath \
  -ldflags "${LDFLAGS}" \
  -tags frpc \
  -o ./release/frpc_windows_amd64_${VERSION}.exe \
  ./cmd/frpc
  ```
  - 从容器中复制编译后的软件到本地 容器外执行
    ```bash
    docker cp KaiFa_exe:/app/release/frpc_windows_amd64_0.69.1.exe ./ts-调试/
    ```
